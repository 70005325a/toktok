# ☁️ 클라우드 배포 — PC 꺼도 24시간 접속되는 영구 주소

내 PC를 꺼도 항상 접속되는 **고정 주소**(`https://이름.onrender.com`)를 무료로 만듭니다.
프로그램은 클라우드(Render)에서 돌고, 친구는 그 주소로 언제나 접속합니다.

> 가입 2개(GitHub, Render)는 본인이 해야 하지만, **git 설치·명령어 없이** 웹에서 클릭/드래그만으로 됩니다.

---

## 1단계 — GitHub에 코드 올리기 (드래그&드롭, git 불필요)

1. https://github.com/signup 에서 무료 가입 (이미 있으면 로그인)
2. 오른쪽 위 **+ → New repository**
   - Repository name: `toktok` (아무거나)
   - **Public** 선택 → **Create repository**
3. 만들어진 페이지에서 **uploading an existing file** 링크 클릭
4. 아래 **파일/폴더만** 드래그해서 올립니다 (나머지는 필요 없음):
   - `server.py`
   - `requirements.txt`
   - `runtime.txt`
   - `render.yaml`
   - `Procfile`
   - `public` **폴더 전체** (안의 index.html, style.css, app.js, sw.js, manifest.webmanifest, icon.svg)
   > ⛔ 올리지 말 것: `cloudflared.exe`, `ngrok.exe`, `*.bat`, `*.ps1` (용량만 차지, 클라우드엔 불필요)
5. 아래 **Commit changes** (초록 버튼) 클릭

> 💡 `public` 폴더째 드래그가 안 되면, 폴더 안 6개 파일을 먼저 올리고, 파일명을 `public/index.html` 처럼 직접 만들어도 됩니다. (보통 폴더 드래그가 됩니다.)

---

## 2단계 — Render에서 배포

1. https://render.com 접속 → **Get Started** → **GitHub로 로그인**(Sign in with GitHub)
2. 대시보드에서 **New +** → **Web Service**
3. 방금 만든 `toktok` 저장소 선택 → **Connect**
4. 설정은 `render.yaml` 덕분에 대부분 자동입니다. 확인만:
   - **Instance Type: Free** 인지 확인
   - (선택) **Environment** 에서 `ACCESS_CODE` 값에 입장 비밀번호 입력
5. **Create Web Service** (또는 Deploy) 클릭
6. 2~4분 빌드 후 상단에 **`https://toktok-xxxx.onrender.com`** 주소가 생깁니다. → 이게 **영구 주소!**

이 주소를 친구에게 보내면, **내 PC와 무관하게 24시간** 접속됩니다.

---

## ⚠️ 무료 등급 특징 (알아두기)
- **15분 동안 아무도 안 들어오면 잠자기** → 다음 첫 접속이 **30~60초** 걸립니다(깨어나는 중). 깨면 그 뒤론 빠름.
- **재배포(코드 다시 올림)하면 대화 기록이 초기화**됩니다 (무료는 영구 디스크 없음). 평소 사용 중엔 유지됩니다.
- 항상 즉시·대화영구보존이 필요하면 유료(월 $7 정도)나 영구 디스크 옵션이 있습니다.

## 업데이트하는 법
GitHub 저장소에서 파일을 다시 올리면(Commit), Render가 자동으로 다시 배포합니다(`autoDeploy`).
